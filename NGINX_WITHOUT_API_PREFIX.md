# Nginx 설정 - /api prefix 없이 사용하기

만약 NestJS에서 `/api` prefix를 사용하지 않고 기존 라우트를 유지하려면, Nginx에서 특정 경로만 백엔드로 프록시하도록 설정할 수 있습니다.

## Nginx 설정 (옵션 2)

        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 백엔드 API 프록시 - questions

        proxy_set_header X-Forwarded-Proto $scheme;

        # CORS 헤더
        add_header 'Access-Control-Allow-Origin' '*' always;
        add_header 'Access-Control-Allow-Methods' 'GET, POST, PUT, DELETE, PATCH, OPTIONS' always;
        add_header 'Access-Control-Allow-Headers' 'Content-Type, Authorization, X-Requested-With' always;
        add_header 'Access-Control-Allow-Credentials' 'true' always;

        if ($request_method = 'OPTIONS') {

## 문제점

이 방식의 단점:
1. **관리가 복잡함** - 새로운 API 엔드포인트 추가 시마다 Nginx 설정 수정 필요
2. **Next.js 라우트와 충돌 가능** - Next.js에서 같은 경로를 사용하면 충돌
3. **설정 중복** - 각 location 블록마다 같은 설정 반복

## 권장사항

**옵션 1 (`/api` prefix 사용)**을 권장합니다:
- 관리가 쉬움
- Next.js 라우트와 명확하게 분리
- 한 번만 설정하면 모든 API에 적용
- 업계 표준 방식
